# 🔊 SoundMaster: 음향기기 통합 관리 시스템

구글 폼즈, 스프레드시트, 그리고 커스텀 대시보드를 결합한 **프리미엄 장비 관리 솔루션**입니다.

## 📁 구성 요소
1.  **Google Forms**: 장비 등록 및 상태 변경 입력용 인터페이스
2.  **Google Sheets**: 데이터 저장소 및 이력 관리
3.  **GAS (Apps Script)**: 시트 데이터를 대시보드로 전달하는 브릿지 API
4.  **Premium Dashboard**: 실시간 현황 파악 및 인벤토리 관리를 위한 고성능 UI

---

## 📖 사용법 및 기능 설명

### 대시보드 (Dashboard)
- 페이지 첫 화면으로, 전체 장비 현황을 한눈에 파악할 수 있습니다.
- 상단 알림 배너: **수리중 · 분실 · 문제발견** 상태의 장비 수를 표시하고, 클릭하면 해당 목록으로 이동합니다.
- 통계 카드: 전체 / 가용 / 대여중 / 수리·문제 장비 수를 실시간으로 집계합니다.
- 차트: 카테고리별 분포(도넛), 부서별 대여 현황(바) 차트를 자동으로 렌더링합니다.
- 우측 상단 **새로고침 버튼**을 누르면 구글 시트에서 최신 데이터를 다시 동기화합니다.

### 인벤토리 (Inventory)
- 등록된 모든 장비를 **목록 보기** 또는 **그리드 보기**로 전환하여 확인할 수 있습니다.
- 기본 보기는 목록 보기이며, 우측 상단 토글 버튼으로 전환합니다.
- **필터 및 검색**: 카테고리 / 위치 / 부서별 필터와 상단 검색창을 조합하여 원하는 장비를 빠르게 찾을 수 있습니다.
- **상태 필터 태그**: 목록 상단의 가용 / 대여중 / 수리중 / 문제발견 / 분실 버튼을 클릭하면 해당 상태 장비만 표시됩니다.
- 장비 카드를 클릭하면 상세 정보 모달이 열립니다.

### 장비 상세 모달
- 장비의 카테고리, 상태, 위치, 사용자, 부서, 목적, 최종 업데이트 날짜를 확인할 수 있습니다.
- **수정 링크**: 해당 장비의 구글 폼 응답 수정 페이지로 바로 이동합니다 (Apps Script가 자동 생성).
- **삭제**: SHA-256 비밀번호 인증 후 장비를 목록에서 제거합니다.

### 메모 기록 시스템 (Notes)
장비 상세 모달 하단의 **"상태 변경 및 메모 기록"** 섹션을 통해 운용 이력을 관리합니다.

| 항목 | 설명 |
|------|------|
| **상태 변경** | 가용 / 대여중 / 수리중 / 문제발견 / 분실 중 선택. 저장 시 구글 시트의 장비 상태도 자동으로 갱신됩니다. |
| **메모** | 문제 사항, 전달 내용 등 자유 텍스트로 기록합니다. |
| **사진 첨부** | 여러 장 첨부 가능. 480px 이하로 자동 압축되며 파일명은 `YYYYMMDD_HHMMSS_원본파일명` 형식으로 저장됩니다. |

- **저장**: 기록은 구글 스프레드시트의 **Notes 시트**와 **PhotosData 시트**에 자동 저장됩니다.
- **조회**: 저장된 기록은 모달 하단 "저장된 기록" 영역에 최신순으로 표시됩니다. 텍스트 메모는 즉시 표시되고, 사진은 백그라운드에서 로드됩니다.
- **삭제**: 각 기록 우측 휴지통 버튼으로 삭제할 수 있으며, 삭제 시 장비 상태는 변경되지 않습니다.

### 통계 (Statistics)
- 상태별 분포(파이), 부서별 장비(레이더), 위치별 장비 수(바) 차트를 제공합니다.

### 데이터 흐름 구조

```
구글 폼 제출
    ↓
설문지 응답 시트1 (장비 등록 · 상태 관리)
    ↓
Apps Script Web App (API 브릿지)
    ├── GET  gviz API → 대시보드 실시간 동기화
    ├── POST addNote  → Notes 시트 + PhotosData 시트 저장
    │                   + 메인 시트 "상태" 컬럼 자동 갱신
    └── POST deleteNote → Notes · PhotosData 삭제 (메인 시트 유지)
```

## 🚀 설정 가이드 (Setup Guide)

### 1단계: 구글 폼 및 시트 준비
1.  **구글 폼 생성**: 다음 항목을 포함하여 만드세요 (드롭다운 추천).
    *   **장비명** (단수 텍스트)
    *   **카테고리** (드롭다운: 스피커, 마이크, 믹서, 케이블, 기타)
    *   **시리얼 번호** (단수 텍스트)
    *   **상태** (드롭다운: 가용, 대여중, 수리중, 분실)
    *   **위치** (드롭다운 필수): 지층 리바이벌 성전, 1층 유치부실, 1층 다윗의 장막, 1층 새가족1실, 1층 새가족2실, 2층 웨일즈성전, 2층 아주사성전, 2층 유아예배실, 2층 영아예배실, 3층 본당, 6층 존웨슬리홀 등
    *   **사용자** (단수 텍스트): 실사용자 이름
    *   **사용 목적** (단수 텍스트): 예) 주일 대예배, 금요철야 등
    *   **사용 부서** (드롭다운): 예배팀, 방송팀, 유치부, 유아부, 영아부, 청년부, 관리팀, 기타
    *   **사용 날짜** (날짜): 대여 또는 사용 시점
2.  **응답 시트 연결**: 폼 응답을 스프레드시트에 저장하도록 설정하세요.

### 2단계: 앱 스크립트(GAS) 설정
1.  스프레드시트에서 `확장 프로그램 > Apps Script` 클릭.
2.  제공된 `appsscript.gs` 파일의 내용을 모두 복사하여 붙여넣기.
3.  상단 `배포 > 새 배포` 클릭.
4.  유형 선택: `웹 앱`
    *   설명: `SoundMaster API`
    *   액세스할 수 있는 사용자: `모든 사용자` (Anonymous 접근 허용 필요)

### 3단계: 대시보드 연결
1.  `script.js` 파일을 열어 상단의 변수를 수정하세요.
    *   `GOOGLE_FORM_URL`: 생성한 구글 폼 URL 입력.
    *   `GOOGLE_SHEET_API`: 구글 폼에 연동된 스프레드시트의 링크 입력.
2.  `index.html`을 실행하면 실시간 데이터가 연동된 대시보드를 확인할 수 있습니다.

### 4단계: 구글 사이트(Google Sites) 임베드
1. 파일 목록 중에 **`google_sites_embed.html`** 파일이 추가되었습니다. 이 파일은 `index.html`, `style.css`, `script.js` 세 개를 구글 사이트용으로 **하나로 합친 최종 통합본**입니다.
2. `google_sites_embed.html` 파일을 메모장이나 텍스트 에디터로 열고, 안의 모든 소스코드를 복사(Ctrl+A, Ctrl+C) 합니다.
3. 구글 사이트(Google Sites) 편집 화면으로 이동합니다.
4. 우측 메뉴에서 `삽입 > 코드 삽입(Embed) > 코드 임베드(Embed code)`를 선택합니다.
5. 복사한 전체 코드를 붙여넣기(Ctrl+V)하고 `다음` -> `삽입`을 클릭합니다.
6. 대시보드 창 크기에 맞게 임베드 위젯의 너비와 높이를 적절히 조절합니다.
7. 우측 상단의 `게시`를 눌러 팀원들에게 공유하세요!
